# 部署说明

## 1. 最低部署要求

- 支持 HTML / CSS / JavaScript 的前端容器
- 可运行在以下环境：
  - WebView
  - Electron
  - H5 容器
  - 浏览器静态页面
  - 设备端前端壳层

## 2. 直接部署

把以下文件原样放入目标软件的静态资源目录：

- `index.html`
- `styles.css`
- `script.js`

然后将页面路由或 WebView 入口指向 `index.html`。

## 3. 嵌入现有软件时的建议

- 如果软件已有统一路由，建议把本目录挂到独立路径，例如：
  - `/modules/child-lock/index.html`
- 如果软件已有父级容器，建议用 iframe 或 WebView 先做无侵入集成，再逐步拆为组件
- 如果软件已有原生能力：
  - “紧急呼叫”按钮可绑定原生拨号
  - “播报警报声”可绑定本地音频
  - “网络设置”可跳原生设置页
  - “设备升级”可跳原生 OTA 页面

## 4. 需要对接的数据点

当前包内已内置演示数据。正式部署时，建议替换以下字段：

- 当前时间
- 当前日期
- 当前熏听资源标题
- 语言/分类信息
- 当前课时
- 资源时长
- 宝宝生日
- 家庭住址
- 紧急联系人

这些字段都集中在 `script.js` 中，便于接入。

## 5. 安全交互约束

- 上滑后不直接解锁，必须进入家长手势验证
- 紧急服务与设置服务都需要长按 2 秒
- 手势错误后自动重置
- 手势成功后仅表现为“临时解锁完成”，便于后续接入真实业务跳转

## 6. 验收建议

部署后至少验证以下流程：

1. 首页显示时间与资源标题
2. 上滑可唤起解锁提示
3. 输入 `1 -> 2 -> 3 -> 6` 可解锁成功
4. 长按左侧按钮 2 秒进入紧急服务
5. 长按右侧按钮 2 秒进入设置服务
6. 紧急服务内可进入宝宝信息卡并返回
